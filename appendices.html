User Manual

The user manual provides detailed guidance for financial executives using the FinSync application to track campaign spending across Chanel's product divisions.

Dashboard Overview

The homepage provides financial executives with a comprehensive overview of campaign spending across Chanel's product lines (F&B, FSH&EW, W&FJ).

Figure 1: Main dashboard interface with budget overview and campaign spending breakdown

Report Generation

To download the report, click the "Download report" button in the bottom right corner of the homepage. This will generate the complete financial report.

Figure 2: Generated report page showing comprehensive financial data

Division-Specific Views

To view reported data for specific divisions, click the corresponding division button (e.g., "F&B", "FSH&EW"). This displays the detailed division report.

Figure 3: F&B division report showing detailed expenditure breakdown

Data Upload

To upload Excel files, click the "Upload" button in the navigation bar. Select "Upload" and choose the document you wish to import into the system.

Figure 4: File upload interface for importing financial data

Deployment Manual

System Requirements: This guide covers deployment of both the Python backend (Azure Functions) and Next.js frontend (Static Web Apps).

Prerequisites

  • Azure account with active subscription
  • Azure CLI installed
  • Node.js (v16+)
  • Python (v3.9+)
  • Next.js project (hybrid rendering)
Step 1: Deploy Azure Functions (Python Backend)
1.1 Install Azure Functions Core Tools
npm install -g azure-functions-core-tools@4
1.2 Initialize a Python Function App
func init MyFunctionApp --python
cd MyFunctionApp
1.3 Create an HTTP Trigger Function
func new --name HttpTrigger --template "HTTP trigger" --authlevel "anonymous"
1.4 Test Locally
func start

Verify at: http://localhost:7071/api/HttpTrigger

1.5 Deploy to Azure
# Log in to Azure
az login

# Create a Resource Group
az group create --name MyResourceGroup --location uksouth

# Create a Storage Account (required for Functions)
az storage account create --name mystorageaccount --location uksouth --resource-group MyResourceGroup --sku Standard_LRS

# Create a Function App
az functionapp create --resource-group MyResourceGroup --consumption-plan-location uksouth --runtime python --runtime-version 3.9 --functions-version 4 --name MyFunctionApp --storage-account mystorageaccount --os-type Linux

# Deploy the code
func azure functionapp publish MyFunctionApp
Step 2: Deploy Next.js Hybrid App (Static Web Apps)
2.1 Configure next.config.js
module.exports = {
  output: "standalone", // Required for Azure Static Web Apps
};
2.2 Initialize SWA CLI
npm install -g @azure/static-web-apps-cli
swa init

Select Hybrid Next.js when prompted.

2.3 Test Locally
npm run build
swa start

Verify at: http://localhost:4280

2.4 Deploy to Azure Static Web Apps
# Log in to Azure (if not already done)
az login

# Create a Static Web App
az staticwebapp create \
  --name MyNextJSApp \
  --resource-group MyResourceGroup \
  --location uksouth \
  --source . \
  --branch main \
  --app-location "." \
  --output-location ".next" \
  --login-with-github # Or use --login-with-azure-ad
2.5 Configure Environment Variables
  1. Go to Azure Portal > Static Web Apps > Configuration
  2. Add:
    • API_URL: https://MyFunctionApp.azurewebsites.net/api/HttpTrigger
Step 3: Secure the Deployment
3.1 Enable HTTPS
az functionapp update --resource-group MyResourceGroup --name MyFunctionApp --https-only true
az staticwebapp update --resource-group MyResourceGroup --name MyNextJSApp --https-only true
3.2 Set Up CORS
az functionapp cors add --resource-group MyResourceGroup --name MyFunctionApp --allowed-origins https://MyNextJSApp.azurestaticapps.net
Step 4: Verify Deployment
  • Frontend: Access at https://MyNextJSApp.azurestaticapps.net
  • Backend: Test API at https://MyFunctionApp.azurewebsites.net/api/HttpTrigger
Troubleshooting

Check logs in Azure Portal > Function App > Logs

Ensure output: "standalone" is set in next.config.js
Note: Replace placeholder names (MyResourceGroup, MyFunctionApp, etc.) with your actual project names.

Legal Issues and Processes

GDPR Compliance

The FinSync project adheres to GDPR regulations (effective May 25, 2018) ensuring proper handling of personal data with strict controls on collection, use, sharing, storage, security, and disposal.

Key Principles
  • Lawfulness, Fairness, Transparency
  • Purpose Limitation
  • Data Minimization
  • Accuracy
  • Storage Limitation
  • Integrity & Confidentiality
Data Subject Rights
  • Right to be Informed
  • Right of Access
  • Right to Rectification
  • Right to Erasure
  • Right to Restrict Processing
  • Right to Data Portability
  • Right to Object
  • Right to Prevent Automated Decision-Making
