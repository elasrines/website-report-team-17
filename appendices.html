Deployment Manual

This guide provides step-by-step instructions for deploying:

  1. Azure Functions (Python backend)
  2. Next.js Hybrid Frontend (Static Web Apps)

Prerequisites

  • Azure account with active subscription
  • Azure CLI installed
  • Node.js (v16+)
  • Python (v3.9+)
  • Next.js project (hybrid rendering)

Step 1: Deploy Azure Functions (Python Backend)

1.1 Install Azure Functions Core Tools
npm install -g azure-functions-core-tools@4
1.2 Initialize a Python Function App
func init MyFunctionApp --python
cd MyFunctionApp
1.3 Create an HTTP Trigger Function
func new --name HttpTrigger --template "HTTP trigger" --authlevel "anonymous"
1.4 Test Locally
func start

Verify at: http://localhost:7071/api/HttpTrigger

1.5 Deploy to Azure
# Log in to Azure
az login

# Create a Resource Group
az group create --name MyResourceGroup --location uksouth

# Create a Storage Account (required for Functions)
az storage account create --name mystorageaccount --location uksouth --resource-group MyResourceGroup --sku Standard_LRS

# Deploy the Function App
az functionapp create --resource-group MyResourceGroup --consumption-plan-location uksouth --runtime python --runtime-version 3.9 --functions-version 4 --name MyFunctionApp --storage-account mystorageaccount --os-type Linux

# Deploy the code
func azure functionapp publish MyFunctionApp

Step 2: Deploy Next.js Hybrid App (Static Web Apps)

2.1 Configure next.config.js for Hybrid Rendering
module.exports = {
  output: "standalone", // Required for Azure Static Web Apps
};
2.2 Initialize SWA CLI
npm install -g @azure/static-web-apps-cli
swa init

Select Hybrid Next.js when prompted.

2.3 Test Locally
npm run build
swa start

Verify at: http://localhost:4280

2.4 Deploy to Azure Static Web Apps
# Log in to Azure (if not already done)
az login

# Create a Static Web App
az staticwebapp create \
  --name MyNextJSApp \
  --resource-group MyResourceGroup \
  --location uksouth \
  --source . \
  --branch main \
  --app-location "." \
  --output-location ".next" \
  --login-with-github # Or use --login-with-azure-ad
2.5 Configure Environment Variables
  1. Go to Azure Portal > Static Web Apps > Configuration
  2. Add:
    • API_URL: https://MyFunctionApp.azurewebsites.net/api/HttpTrigger

Step 3: Secure the Deployment

3.1 Enable HTTPS
az functionapp update --resource-group MyResourceGroup --name MyFunctionApp --https-only true
az staticwebapp update --resource-group MyResourceGroup --name MyNextJSApp --https-only true
3.2 Set Up CORS (For Frontend-Backend Communication)
az functionapp cors add --resource-group MyResourceGroup --name MyFunctionApp --allowed-origins https://MyNextJSApp.azurestaticapps.net

Step 4: Verify Deployment

  1. Frontend: Access at https://MyNextJSApp.azurestaticapps.net
  2. Backend: Test API at https://MyFunctionApp.azurewebsites.net/api/HttpTrigger

Troubleshooting

  • Function App Fails to Start: Check logs in Azure Portal > Function App > Logs.
  • Static Web App 404 Errors: Ensure output: "standalone" is set in next.config.js.

Final Notes

  • Replace placeholder names (MyResourceGroup, MyFunctionApp, etc.) with your project names.
  • For detailed logs, refer to:
    • Azure Functions Docs
    • Static Web Apps Docs

Legal Issues and Processes

This section covers the legal considerations and processes related to the FinSync project. It includes information on data privacy, compliance, intellectual property, and GDPR (General Data Protection Regulation).

Data Privacy and GDPR

The General Data Protection Regulation (GDPR) came into effect on 25th May 2018, replacing the Data Protection Act. It increases privacy rights for individuals and imposes stricter controls on the collection, use, sharing, storage, security, and disposal of personal data. All organisations must state what information they hold about individuals, and no personal data can be stored outside the UK without proper safeguards.

Key GDPR Principles:
  • Lawfulness, Fairness, and Transparency: Data must be processed lawfully, fairly, and in a transparent manner.
  • Purpose Limitation: Data should be collected for specified, explicit, and legitimate purposes and not further processed in a manner incompatible with those purposes.
  • Data Minimization: Data should be adequate, relevant, and limited to what is necessary for the intended purpose.
  • Accuracy: Data must be accurate and kept up to date.
  • Storage Limitation: Data should be kept in a form that permits identification of data subjects for no longer than necessary.
  • Integrity and Confidentiality: Data must be processed securely, protecting against unauthorized or unlawful processing, accidental loss, destruction, or damage.
Data Subject Rights:

Under GDPR, individuals have the following rights:

  • Right to be Informed: Individuals must be informed about how their data is being used.
  • Right of Access: Individuals can request access to their personal data.
  • Right to Rectification: Individuals can request corrections to inaccurate or incomplete data.
  • Right to Erasure: Individuals can request the deletion of their data under certain circumstances.
  • Right to Restrict Processing: Individuals can request the restriction of data processing under certain conditions.
  • Right to Data Portability: Individuals can request their data in a structured, commonly used format.
  • Right to Object: Individuals can object to the processing of their data for specific purposes.
  • Right to Prevent Automated Decision-Making: Individuals can request human intervention in automated decision-making processes.
Privacy Notice Policy

A Privacy Notice Policy must be provided to data subjects whenever new processing is undertaken. It should include:

  • Important information about the data controller.
  • Details of the data collected, how it is collected, and how it is used.
  • Information on data security, retention, and international transfers.
  • Explanation of individuals' legal rights under GDPR.

Intellectual Property (IP) and Source Code License

Intellectual property (IP) is a critical consideration in software development. The FinSync project ensures that all IP rights are respected, and the source code is licensed appropriately. The following points outline our approach to IP and licensing:

  • Copyright: The source code and other creative works produced during the project are protected under copyright law. The copyright holder has exclusive rights to reproduce, distribute, and adapt the work.
  • Source Code License: The source code for the FinSync project is licensed under a closed-source license. The software remains the property of the the company.
  • Third-Party Components: Any third-party libraries or components used in the project are properly attributed and comply with their respective licenses.
